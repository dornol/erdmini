# 기능추가 및 이슈에 대한 낙서

## 기능추가 검토

### 다수의 사용자 지원 (보류)
프로젝트 철학이 mini이고, ERD 도구 특성상 동시 사용자가 많지 않음.
현재 구조(SQLite WAL + 단일 Node.js WebSocket)로 동시 접속 50~100명, 동시 편집 10~20명 수준은 충분히 감당 가능.
직원 500명 규모에서도 동시 사용자 10명 내외로 추정되어, 당분간 현행 유지.

단, MCP 연동이 활발해지면 사용 패턴이 달라질 수 있음:
- AI 코딩 도구(Claude Code, Cursor 등)가 스키마 조회/수정을 자동 호출 → 요청 빈도 증가
- CI/CD 스키마 lint, AI 마이그레이션 생성 시 스키마 참조 등
- 다만 MCP는 가벼운 REST 호출이라 WebSocket 동시 접속과 성격이 다름
- SQLite 읽기는 사실상 무제한, 단건 쓰기 부하도 미미 → 현재 구조로 충분

수천 명 규모 조직에서 필요해질 경우 검토 사항:
- 외부 DBMS (PostgreSQL 등) 전환: 동시 쓰기 병목 해소
- Redis 도입: 세션 공유, WebSocket pub/sub (서버 이중화 시)
- 로드밸런서 + 다중 인스턴스 구성
